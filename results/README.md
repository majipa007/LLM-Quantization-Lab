# `results/`

Everything the benchmarking stages produce: summary CSVs at the top level, and
raw logs / metadata in subfolders.

## Summary files

| File | Contents | Created by |
| --- | --- | --- |
| `size_results.csv` | File size (bytes/MiB/GiB) and % reduction vs BF16 for each GGUF. | `scripts/05` |
| `speed_results.csv` | Prompt-processing and text-generation throughput (tokens/s) for each GGUF. | `scripts/06` |
| `perplexity_results.csv` | WikiText-2 perplexity and uncertainty for each GGUF, with a status column. | `scripts/08` |
| `SHA256SUMS` | SHA-256 checksum of every GGUF. | `scripts/11` |
| `failed_quantizations.txt` | Any quant formats that failed (only present if some failed). | `scripts/04` |

## Subfolders

| Path | Contents | Created by |
| --- | --- | --- |
| `raw/quantization/` | Per-format quantization logs and the exact command used. | `scripts/04` |
| `raw/speed/` | Raw `llama-bench` JSON output per format. | `scripts/06` |
| `raw/perplexity/` | Raw `llama-perplexity` logs per format. | `scripts/08` |
| `raw/generations/<format>/` | Generated text samples, one file per prompt. | `scripts/09` |
| `raw/convert.log` | Log of the HF→GGUF conversion. | `scripts/03` |
| `metadata/` | Commit hashes, tool versions, corpus path and the full hardware + benchmark configuration. | `scripts/01`, `02`, `10` |

The CSV summaries are small and worth committing; the large raw logs and
generations can be regenerated by re-running the pipeline.
